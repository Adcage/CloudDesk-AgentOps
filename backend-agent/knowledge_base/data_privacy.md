# CloudDesk 数据隐私政策

## 1. 概述

本政策规定了 CloudDesk 平台在数据收集、处理、存储和删除方面的隐私保护要求，确保符合欧盟《通用数据保护条例》（GDPR）及其他适用法律法规。

## 2. 数据收集与处理

### 2.1 收集的数据类型

| 数据类型 | 具体内容 | 收集目的 | 法律依据 |
|---------|---------|---------|---------|
| 身份数据 | 姓名、邮箱、手机号 | 账号管理、身份验证 | 合同履行 |
| 使用数据 | 登录时间、操作记录、对话内容 | 服务提供、质量改进 | 合同履行 |
| 支付数据 | 信用卡末 4 位、支付记录 | 费用结算 | 合同履行 |
| 技术数据 | IP 地址、浏览器信息、设备信息 | 安全防护、问题排查 | 合法利益 |
| Cookie 数据 | 会话标识、偏好设置 | 服务运营 | 同意/合法利益 |

### 2.2 数据处理原则

1. **合法、公正、透明**：仅出于指定目的处理数据，并告知用户
2. **目的限制**：不将数据用于与收集目的无关的处理
3. **数据最小化**：仅收集实现目的所必需的最少数据
4. **准确性**：确保数据准确且及时更新
5. **存储限制**：数据保存不超过必要期限
6. **完整性和保密性**：采取适当的技术和组织措施保护数据

### 2.3 数据处理者与控制者

- CloudDesk 作为**数据处理者**处理企业客户的最终用户数据
- 企业客户作为**数据控制者**决定其用户数据的处理目的和方式
- CloudDesk 与企业客户签订 DPA（数据处理协议）
- CloudDesk 不出售用户数据给第三方

## 3. 数据删除请求处理

### 3.1 删除权（Right to Erasure）

用户有权要求删除其个人数据，CloudDesk 按以下流程处理：

**步骤 1：验证身份**
- 用户提交删除请求时验证身份（邮箱验证 + 两步验证）
- 企业用户需通过企业管理员提交删除请求

**步骤 2：评估删除范围**

| 数据类型 | 是否删除 | 说明 |
|---------|---------|------|
| 对话记录 | 是 | 删除 agent_messages 中的相关记录 |
| 个人信息 | 是 | 匿名化 business.customers 中的个人标识 |
| 工单记录 | 保留 | 匿名化后保留用于统计，删除个人标识 |
| 审计日志 | 保留 | 法律要求保留，但匿名化处理 |
| 支付记录 | 保留 | 财务合规要求保留 5 年，但匿名化 |
| 聚合数据 | 保留 | 不包含个人信息的统计数据 |

**步骤 3：执行删除**
- 自删除请求确认后 30 天内完成数据删除
- 删除操作记入审计日志
- 删除完成后通知用户

**步骤 4：验证与通知**
- 系统自动验证数据已不包含用户个人标识
- 向用户发送删除确认通知
- 如因法律原因无法删除全部数据，说明原因和保留期限

### 3.2 删除请求例外

以下情况可能拒绝或延迟数据删除请求：

- 法律要求保留的数据（财务记录、审计日志等）
- 正在进行的争议或调查相关的数据
- 删除将影响其他用户数据完整性的情况
- 企业管理员要求保留的员工数据（需告知数据主体）

### 3.3 企业客户数据删除

Enterprise 客户可请求批量删除其最终用户数据：

1. 提交批量删除请求（包含用户列表或全部用户选项）
2. CloudDesk 验证企业管理员身份
3. 确认删除范围和时间安排
4. 按约定时间执行批量删除
5. 提供 deletion certificate 作为删除证明

## 4. GDPR 合规

### 4.1 合规框架

- CloudDesk 总部位于欧盟区，遵守 GDPR 所有条款
- 已指定数据保护官（DPO）：dpo@clouddesk.com
- 与 EU 数据保护机构保持沟通
- 每年进行 GDPR 合规性审计

### 4.2 数据主体权利

| 权利 | CloudDesk 支持 | 处理时限 |
|------|-------------|---------|
| 访问权 (Right of Access) | 是 | 30 天 |
| 更正权 (Right to Rectification) | 是 | 30 天 |
| 删除权 (Right to Erasure) | 是 | 30 天 |
| 限制处理权 (Right to Restriction) | 是 | 30 天 |
| 数据可携带权 (Right to Portability) | 是 | 30 天 |
| 反对权 (Right to Object) | 是 | 30 天 |

### 4.3 数据处理协议（DPA）

- CloudDesk 提供标准 DPA 模板
- Enterprise 客户可要求签订定制 DPA
- DPA 包含以下核心条款：
  - 数据处理目的和范围
  - 数据分类和标记
  - 子处理者管理
  - 安全措施要求
  - 数据泄露通知流程
  - 数据删除和返还条款

### 4.4 国际数据传输

- EU/EEA 内的用户数据存储在欧盟区数据中心
- 如需向境外传输，采用以下保障措施：
  - 签署欧盟标准合同条款（SCCs）
  - 通过数据保护影响评估（DPIA）
  - 获得数据主体明确同意
- CloudDesk 不在用户不知情的情况下跨境传输数据

### 4.5 数据保护影响评估（DPIA）

- 新功能上线前进行 DPIA 评估
- DPIA 包含：
  - 数据处理目的描述
  - 数据流分析
  - 隐私风险评估
  - 缓解措施
  - 残余风险接受声明

## 5. 数据保留期限

### 5.1 保留期限表

| 数据类型 | 保留期限 | 到期处理 |
|---------|---------|---------|
| 用户账号信息 | 账号存续期 + 30 天 | 匿名化 |
| 对话记录 | 账号存续期 + 90 天 | 删除 |
| 工单记录 | 账号存续期 + 1 年 | 匿名化 |
| 支付记录 | 5 年（财务合规） | 匿名化 |
| 审计日志 | 2 年 | 归档至冷存储 |
| AI 训练数据 | 不使用用户数据训练 AI | N/A |
| 聚合统计数据 | 无限期 | 不含个人信息 |
| 会话数据 | 会话结束后 30 天 | 删除 |

### 5.2 保留期限例外

- 法律诉讼中的数据：保留至诉讼终结
- 监管调查中的数据：保留至调查终结
- Enterprise 合同约定的保留期限优先

### 5.3 数据匿名化

匿名化处理遵循以下标准：

- 移除所有直接标识信息（姓名、邮箱、手机号、IP 地址）
- 移除或泛化间接标识信息（替换为范围值、去掉精确时间戳）
- 确保匿名化后不能通过合理方式重新识别个人
- 匿名化数据不再视为个人数据，可用于统计分析

## 6. 数据安全技术措施

### 6.1 加密

| 场景 | 加密方式 |
|------|---------|
| 传输加密 | TLS 1.3 |
| 存储加密 | AES-256 |
| 密码存储 | bcrypt (cost factor = 12) |
| 数据库备份加密 | AES-256 |
| 向量数据 | 不额外加密（性能考虑） |

### 6.2 访问控制

- 最小权限原则：员工仅访问完成工作所需的最少数据
- 基于角色的访问控制（RBAC）
- 管理员操作需两步验证
- 数据访问操作记录在审计日志中
- 定期审查访问权限（每季度）

### 6.3 数据泄露响应

| 步骤 | 时限 | 责任方 |
|------|------|--------|
| 发现与确认 | 2 小时 | 安全团队 |
| 评估影响范围 | 12 小时 | 安全团队 + DPO |
| 通知监管机构 | 72 小时 | DPO |
| 通知受影响用户 | 72 小时 | 客服团队 |
| 修复和补救 | 7 天 | 安全团队 + 工程 |
| 事后复盘 | 30 天 | 安全团队 + 管理 |

## 7. 用户数据导出

### 7.1 数据可携带权

用户可请求导出其个人数据：

- 导出格式：JSON 或 CSV
- 导出内容：用户信息、对话记录、工单记录、支付记录
- 导出方式：下载链接（有效期 7 天）
- 处理时限：30 天内

### 7.2 企业客户数据导出

Enterprise 客户可随时通过控制台批量导出数据：

- 支持全量数据导出
- 支持按日期范围筛选导出
- 支持增量导出（仅导出自上次导出后的变更数据）
- 导出文件加密后提供下载

## 8. Cookie 和追踪

### 8.1 Cookie 类型

| 类型 | 用途 | 是否必须 | 保留期 |
|------|------|---------|--------|
| 会话 Cookie | 维持登录状态 | 是 | 会话期 |
| 偏好 Cookie | 存储用户偏好设置 | 否 | 1 年 |
| 分析 Cookie | 网站使用分析 | 否 | 2 年 |
| 营销 Cookie | 广告和再营销 | 否 | 1 年 |

### 8.2 Cookie 同意

- Free 和 Business 用户：首次访问时展示 Cookie 同意弹窗
- Enterprise 用户：可在控制台配置 Cookie 策略
- 用户可随时在设置中修改 Cookie 偏好
