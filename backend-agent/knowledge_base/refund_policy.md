# CloudDesk 退款政策

## 1. 政策概述

CloudDesk 退款政策适用于所有通过 CloudDesk 平台购买的订阅服务。本政策旨在明确退款条件、流程和审批规则，确保客户权益得到合理保障的同时维护平台运营秩序。

所有退款请求均通过客服智能体提交，由系统自动判断是否需要人工审批。

## 2. 可退款场景

### 2.1 重复扣费

当客户账户因系统故障或技术错误被收取多次相同款项时，客户有权申请全额退款。重复扣费的判定标准如下：

- 同一客户、同一订阅周期、相同金额的扣款记录出现两次或以上
- 扣款时间间隔在 72 小时以内
- 订阅状态在扣款时间内处于活跃状态

重复扣费退款处理时间：1-3 个工作日，无需审批。

### 2.2 服务未生效

客户在以下情况可申请全额退款：

- 订阅支付成功后 7 个自然日内，服务尚未开通或无法正常使用
- 订阅续费后，功能权限未在 24 小时内更新
- 套餐升级后，新功能在 48 小时内仍未可用
- 因平台原因导致服务中断超过 24 小时

服务未生效退款处理时间：3-5 个工作日，无需审批（金额低于 $100 时）。

### 2.3 多收费

当客户实际被收取的金额与订阅计划标价不符时，客户可以申请退还差额或多收的部分：

- 订阅价格变更后，仍按旧价格扣款的情况
- 促销折扣未正确应用导致多收费
- 套餐降级后仍按原套餐价格扣款
- 税费计算错误导致的额外收费

多收费退款处理时间：3-5 个工作日，无需审批（金额低于 $100 时）。

### 2.4 试用期误扣费

- 客户在免费试用期内被扣费，可申请全额退款
- 试用到期后未通知即扣费，退还首次扣款金额
- 处理时间：1-2 个工作日

### 2.5 年度套餐中途退款

- Business 和 Enterprise 年度套餐客户可在购买后 30 个自然日内申请按比例退款
- 退款金额 = 已支付金额 - (已使用天数 / 365 × 日均费用)
- 日均费用 = 年度总费用 / 365
- 年度套餐退款需审批

## 3. 不可退款场景

### 3.1 超过退款时效

- 退款申请距离扣款日期超过 30 个自然日（特殊场景除外）
- 年度套餐超过购买后 30 天的退款窗口期
- 试用期扣费超过 60 天的退款申请

### 3.2 已使用服务

以下情况视为服务已充分使用，不支持退款：

- 订阅使用超过一个完整计费周期（月度套餐）
- 在订阅期内已提交超过 50 个工单（Free 套餐除外）
- 已使用 Enterprise 专属客户经理服务
- 已触发 SLA 补偿条款的订阅

### 3.3 违反使用条款

- 因违反 CloudDesk 服务条款导致账户被冻结的
- 因滥用平台资源被终止服务的
- 涉及欺诈行为的交易

### 3.4 其他不可退款情况

- Free 套餐本身无付费项目，不涉及退款
- 通过促销活动获得的免费使用时长不可折现退款
- 第三方支付渠道的手续费部分不予退还
- 货币兑换导致的汇率差额不予补偿

## 4. 退款审批规则

### 4.1 审批阈值

| 条件 | 是否需要审批 |
|---|---|
| 退款金额 < $100 | 不需要审批，自动处理 |
| 退款金额 >= $100 | 需要主管审批 |
| 退款金额 >= $500 | 需要 Senior Manager 审批 |
| 退款金额 >= $1000 | 需要 VP 级别审批 |

### 4.2 审批流程

1. 客服智能体判断退款请求需要审批
2. 系统自动创建审批单，包含以下信息：
   - 客户 ID 和订单 ID
   - 退款金额和退款原因
   - 相关凭证（重复扣费记录、服务中断日志等）
3. 审批单状态为 pending，等待管理人员审核
4. 管理人员可在 Approvals 页面进行 Approve 或 Reject 操作
5. 审批通过后，退款在 5-7 个工作日内到账
6. 审批被驳回的，客户可通过工单系统申诉

### 4.3 审批时限

- 一级审批（$100-$500）：24 小时内完成
- 二级审批（$500-$1000）：48 小时内完成
- 三级审批（$1000 以上）：72 小时内完成

### 4.4 加急审批

- Enterprise 客户的退款审批自动加急标记
- 涉及客户留存风险的退款可申请加急
- 加急审批应在正常时限的一半内完成

## 5. 退款流程

### 5.1 标准退款流程

```
客户发起请求 → 智能体意图识别 → 判断退款类型 → 计算退款金额
→ 检查审批阈值 → （需要审批？） → 自动退款 / 创建审批单
→ 通知客户退款状态 → 退款完成确认
```

### 5.2 退款方式

- 原路退回：退款至原支付方式（信用卡、银行转账等）
- 账户余额：退款至客户 CloudDesk 账户余额（立即到账）
- 客户可选择退款方式，默认原路退回

### 5.3 退款凭证

所有退款操作均需要以下凭证之一：

- 支付截图或银行对账单
- 系统自动生成的扣费记录
- 客服智能体的工单摘要作为退款依据

### 5.4 退款状态跟踪

- pending：退款申请已提交，待处理
- processing：退款正在处理中
- completed：退款已完成
- rejected：退款申请被驳回
- cancelled：客户取消退款申请

## 6. 特殊场景处理

### 6.1 企业客户退款

- Enterprise 客户享有优先退款处理通道
- 退款审批时限缩短 50%
- 可指定专属客户经理协助处理退款事宜
- 支持定制化退款方案（如部分退款转为服务延期）

### 6.2 批量退款

- 因系统故障导致的批量扣费问题，自动启动批量退款流程
- 批量退款无需客户逐个申请
- 受影响客户将通过邮件和站内信通知
- 退款将在 3 个工作日内统一处理

### 6.3 争议退款

- 客户对退款金额有异议的，可申请争议处理
- 争议处理需提交详细说明和相关凭证
- 争议退款由 Senior Manager 级别以上审批
- 争议处理时限为 5 个工作日

### 6.4 跨境退款

- 跨境退款可能因汇率波动产生差额，以退款当日汇率为准
- 跨境退款处理时间为 5-10 个工作日
- 银行手续费由客户承担

## 7. 退款记录与审计

- 所有退款操作均记录在审计日志中
- 退款记录保留期限为 3 年
- 支持按客户 ID、订单 ID、退款时间范围查询退款记录
- 退款统计包含在月度财务报表中

## 8. 政策更新

- 退款政策更新将通过平台公告和邮件通知所有客户
- 政策更新后有 30 天过渡期，过渡期内按旧政策执行
- 重大政策变更需征询 Enterprise 客户意见

## 9. 常见问题

### Q1: 退款后订阅状态如何变化？
A: 退款完成后，订阅将根据退款类型决定状态。全额退款将导致订阅降级或取消；部分退款不影响当前订阅状态。

### Q2: 多次小额退款是否需要审批？
A: 单笔退款金额低于 $100 不需要审批。但如果同一客户在 30 天内累计退款超过 $300，系统将自动触发审批流程。

### Q3: 退款被驳回怎么办？
A: 客户可通过工单系统提交申诉，附上额外凭证。申诉将由更高级别管理人员审核。

### Q4: 退款到账时间？
A: 原路退回通常需要 5-7 个工作日（跨境可能更长）；账户余额退款立即到账。
