# CloudDesk 登录问题排查指南

## 1. 概述

本文档提供 CloudDesk 平台登录相关问题的全面排查步骤和解决方案，涵盖密码重置、账号锁定、两步验证和 SSO 集成等常见场景。

## 2. 常见登录问题分类

### 2.1 问题类型分布

| 问题类型 | 占比 | 优先级 | 平均处理时间 |
|---------|------|--------|-----------|
| 忘记密码 | 45% | P3 | 5 分钟 |
| 账号锁定 | 25% | P2 | 15 分钟 |
| 两步验证问题 | 15% | P2 | 20 分钟 |
| SSO 登录失败 | 10% | P1 | 30 分钟 |
| 其他 | 5% | P3 | 10 分钟 |

## 3. 密码重置

### 3.1 用户自助重置

用户可通过以下步骤自行重置密码：

1. 在登录页面点击"忘记密码"链接
2. 输入注册邮箱地址
3. 系统发送重置密码邮件到注册邮箱
4. 邮件包含一次性重置链接，有效期 30 分钟
5. 点击链接进入重置密码页面
6. 设置新密码（需满足密码策略要求，见第 5 节）
7. 重置成功后自动跳转至登录页面

### 3.2 重置链接失效处理

| 情况 | 解决方案 |
|------|---------|
| 超过 30 分钟过期 | 重新发起密码重置请求 |
| 邮件未收到 | 检查垃圾邮箱；确认注册邮箱正确；联系客服 |
| 链接已使用 | 每个重置链接只能使用一次，需重新发起 |
| 多次未收到邮件 | 检查邮件服务商是否屏蔽 CloudDesk 域名 |

### 3.3 客服协助重置

当用户无法自助重置密码时（如注册邮箱不可用），客服可协助重置：

1. 验证用户身份（至少提供 3 项：注册邮箱、手机号、最近登录 IP、账户创建日期）
2. 在系统中生成管理员重置码
3. 通过安全渠道（电话或安全邮箱）将重置码告知用户
4. 用户使用重置码在专门页面设置新密码
5. 重置操作记入审计日志（business.audit_logs）
6. 系统自动发送密码重置通知邮件

### 3.4 管理员强制重置

企业管理员可强制重置团队成员密码：

1. 管理员登录控制台，进入"团队管理"页面
2. 选择需重置密码的成员
3. 点击"重置密码"按钮
4. 系统生成临时密码并发送至成员邮箱
5. 成员首次使用临时密码登录后必须修改密码

## 4. 账号锁定

### 4.1 锁定规则

| 规则 | 参数 |
|------|------|
| 连续错误登录次数 | 5 次 |
| 锁定时长 | 30 分钟 |
| 锁定后解锁方式 | 自动解锁 / 管理员手动解锁 |
| 连续锁定次数上限 | 3 次（第 3 次后需人工验证解锁） |

### 4.2 自动锁定流程

1. 系统记录每次登录失败的时间、IP 和设备信息
2. 连续第 5 次失败后自动锁定账号
3. 系统发送锁定通知邮件至注册邮箱
4. 30 分钟后自动解锁（如未触发第 4.3 节规则）
5. 解锁后再次错误登录 5 次，重新锁定 30 分钟

### 4.3 疑似入侵锁定

如果检测到以下情况，账号将被永久锁定（需人工解锁）：

- 短时间内从 3 个以上不同 IP 尝试登录
- 连续 3 次触发 30 分钟自动锁定
- 已知泄露密码列表中的密码被用于尝试登录
- 使用字典攻击或暴力破解特征的登录模式

### 4.4 解锁流程

**自助解锁（30 分钟自动锁定）：**
1. 等待 30 分钟自动解锁
2. 或使用"忘记密码"功能重置密码（重置后自动解锁）

**人工解锁（疑似入侵锁定）：**
1. 用户联系客服
2. 客服验证用户身份（详见账号安全文档）
3. 客服在系统中执行手动解锁操作
4. 强制用户下次登录时修改密码
5. 解锁操作记入审计日志

### 4.5 企业管理员解锁

企业管理员可在"团队管理"页面直接解锁被锁定的成员账号。解锁后系统通知相关成员，并建议修改密码。

## 5. 两步验证（2FA）

### 5.1 两步验证类型

| 类型 | 说明 | 安全等级 |
|------|------|---------|
| TOTP（基于时间的一次性密码） | 使用 Google Authenticator 等应用生成验证码 | 高 |
| 短信验证码 | 发送 6 位数字验证码到绑定手机 | 中 |
| 邮箱验证码 | 发送 6 位数字验证码到注册邮箱 | 中 |

### 5.2 启用两步验证

1. 登录控制台，进入"账号安全"设置
2. 选择"启用两步验证"
3. 选择验证方式（推荐 TOTP）
4. 扫描二维码绑定验证应用
5. 输入验证应用显示的验证码确认绑定
6. 系统生成备用恢复码（共 10 个，建议保存到安全位置）

### 5.3 两步验证问题排查

**问题：丢失验证设备**

1. 在登录页面点击"无法使用验证器"
2. 输入备用恢复码之一（每个恢复码只能使用一次）
3. 登录成功后进入安全设置重新绑定验证设备
4. 如果恢复码也丢失，需联系客服进行身份验证后重置

**问题：验证码始终不正确**

1. 确认设备时间同步正确（TOTP 依赖精确时间）
2. 尝试重新扫描二维码绑定
3. 检查验证应用是否支持 SHA-256 算法
4. 如问题持续，联系客服重置两步验证

**问题：短信验证码未收到**

1. 确认绑定手机号正确
2. 检查手机信号和短信拦截设置
3. 等待 60 秒后重新发送
4. 连续 3 次未收到可切换为邮箱验证码
5. 如手机号已变更，联系客服协助更新

### 5.4 强制两步验证

- Business 套餐管理员可设置团队级别的两步验证策略
- 策略选项：可选 / 推荐 / 强制
- 设为"强制"后，团队成员必须在 7 天内启用两步验证，否则账号被停用
- Enterprise 套餐支持与 IdP 集成，统一身份认证

## 6. SSO（单点登录）集成

### 6.1 支持的 SSO 协议

| 协议 | 版本 | 适用套餐 |
|------|------|---------|
| SAML 2.0 | 2.0 | Enterprise |
| OAuth 2.0 | 2.0 | Business |
| OpenID Connect | 1.0 | Enterprise |

### 6.2 SSO 配置步骤

1. 管理员在 CloudDesk 控制台进入"集成管理"→"SSO 配置"
2. 选择 SSO 协议类型
3. 填写 IdP（身份提供者）信息：
   - IdP 名称
   - Entity ID / Client ID
   - SSO URL / Authorization URL
   - Certificate / Client Secret
   - 用户属性映射（邮箱、姓名、角色等）
4. 测试连接
5. 保存并启用

### 6.3 SSO 登录问题排查

**问题：SSO 登录后跳转错误**

1. 检查 IdP 配置的回调 URL 是否正确（应为 https://app.clouddesk.com/auth/sso/callback）
2. 确认 IdP 的 Entity ID 与 CloudDesk 配置一致
3. 检查 SSL 证书是否有效
4. 清除浏览器缓存后重试

**问题：SSO 登录失败 - 认证错误**

1. 检查 IdP 和 CloudDesk 之间的时钟同步（SAML 断言有效期窗口 ±5 分钟）
2. 确认 SAML 断言签名算法是否匹配
3. 验证用户在 IdP 中的属性映射是否正确
4. 查看 IdP 日志和 CloudDesk SSO 日志

**问题：用户角色不正确**

1. 检查属性映射中角色字段的配置
2. 确认 IdP 发送的角色值与 CloudDesk 角色对应关系：
   - IdP 角色 "admin" → CloudDesk 角色 "admin"
   - IdP 角色 "manager" → CloudDesk 角色 "manager"
   - IdP 角色 "user" → CloudDesk 角色 "support_agent"
3. 如角色映射不符合预期，修改 SSO 配置中的属性映射规则

### 6.4 SSO 故障降级

- 当 SSO 服务不可用时，管理员可临时切换为 CloudDesk 本地认证模式
- 切换后 SSO 用户可使用本地密码登录
- 已启用两步验证的用户仍需通过两步验证
- SSO 恢复后管理员重新启用 SSO 模式

## 7. 其他登录问题

### 7.1 浏览器兼容性

| 浏览器 | 最低版本 | 推荐版本 |
|--------|---------|---------|
| Chrome | 90+ | 最新版 |
| Firefox | 88+ | 最新版 |
| Safari | 14+ | 最新版 |
| Edge | 90+ | 最新版 |

不支持 IE11 及以下版本。

### 7.2 Cookie 问题

- CloudDesk 登录依赖 Cookie，需确保浏览器未禁用 Cookie
- 如使用隐私浏览模式，可能需要每次重新登录
- 推荐将 clouddesk.com 加入浏览器信任站点列表

### 7.3 会话超时

| 套餐 | 会话超时 | 可配置 |
|------|---------|--------|
| Free | 30 分钟 | 否 |
| Business | 60 分钟 | 是（15-240 分钟） |
| Enterprise | 自定义 | 是（5-1440 分钟） |

### 7.4 多设备登录

- Free 套餐：仅允许 1 个设备同时在线
- Business 套餐：最多 3 个设备同时在线
- Enterprise 套餐：无限制
- 超过上限时最早登录的设备被自动踢出，并收到通知
