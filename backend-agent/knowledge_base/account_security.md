# CloudDesk 账号安全政策

## 1. 概述

本政策定义了 CloudDesk 平台账号的安全管理规则，包含账号冻结、密码策略、会话管理和风险等级定义。所有 CloudDesk 用户和客服人员必须遵守本政策。

## 2. 账号冻结流程

### 2.1 冻结触发条件

账号将在以下情况下被自动冻结：

| 触发条件 | 冻结类型 | 持续时间 |
|---------|---------|---------|
| 连续 5 次登录失败 | 临时锁定 | 30 分钟 |
| 疑似入侵行为检测 | 安全冻结 | 需人工解锁 |
| 管理员主动操作 | 管理冻结 | 直到管理员解冻 |
| 违反服务条款 | 合规冻结 | 需合规审核解除 |
| 逾期未付款超过 30 天 | 财务冻结 | 付款后自动解冻 |

### 2.2 冻结流程

```
触发冻结条件
    ↓
系统自动冻结账号（status = frozen）
    ↓
发送通知邮件给注册邮箱
    ↓
如果是临时锁定 → 30 分钟后自动解冻
如果是安全冻结 → 通知客服和安全团队
    ↓
客服审核并确认解冻条件
    ↓
解冻操作（status = active）→ 记录审计日志
```

### 2.3 解冻流程

**自助解冻（临时锁定）：**
- 等待 30 分钟自动解冻
- 使用"忘记密码"功能重置密码后自动解冻

**人工解冻（安全冻结 / 管理冻结 / 合规冻结）：**
1. 用户联系客服提交解冻申请
2. 客服验证用户身份（见第 2.4 节）
3. 客服在系统中执行解冻操作，将 customer.status 从"frozen"改为"active"
4. 系统强制用户下次登录时修改密码
5. 解冻操作记入 business.audit_logs

### 2.4 用户身份验证

解冻时需验证用户身份，至少满足以下 3 项：

| 验证项 | 说明 |
|--------|------|
| 注册邮箱 | 能接收验证码 |
| 手机号码 | 能接收短信验证码 |
| 最近登录 IP | 与历史记录比对 |
| 账户创建日期 | 用户需准确提供 |
| 最近操作描述 | 描述最近一次使用 CloudDesk 的操作 |
| 安全问答 | 用户预设的安全问题（仅 Business 和 Enterprise） |

 Enterprise 客户需额外验证企业域名邮箱。

## 3. 密码策略

### 3.1 密码复杂度要求

| 规则 | Free | Business | Enterprise |
|------|------|----------|-----------|
| 最小长度 | 8 字符 | 10 字符 | 12 字符 |
| 必须包含大写字母 | 否 | 是 | 是 |
| 必须包含小写字母 | 是 | 是 | 是 |
| 必须包含数字 | 是 | 是 | 是 |
| 必须包含特殊字符 | 否 | 是 | 是 |
| 不允许与用户名相同 | 是 | 是 | 是 |
| 不允许与最近 N 次密码相同 | 3 次 | 5 次 | 10 次 |
| 密码过期周期 | 无 | 90 天 | 60 天 |

### 3.2 密码存储

- 密码使用 bcrypt 算法加密存储（cost factor = 12）
- 数据库中不存储明文密码
- 密码哈希使用独立盐值
- 历史密码仅存储哈希值用于重复检测

### 3.3 密码修改

- 用户可在"账号安全"页面主动修改密码
- 修改密码需验证当前密码
- 新密码需满足复杂度要求且不与最近 N 次密码相同
- 修改成功后所有设备会话失效，需重新登录
- 密码修改操作记入审计日志

### 3.4 初始密码

- 新创建账号的初始密码通过安全渠道发送
- 首次登录必须修改初始密码
- 初始密码有效期 7 天，超期未使用需重新生成

## 4. 会话管理

### 4.1 会话创建

- 用户登录成功后创建会话（agent_sessions）
- 会话关联唯一 session_id 和 conversation_id
- 会话信息包括：用户 ID、当前 Agent、登录时间、IP 地址

### 4.2 会话超时配置

| 套餐 | 空闲超时 | 绝对超时 | 可配置 |
|------|---------|---------|--------|
| Free | 30 分钟 | 8 小时 | 否 |
| Business | 60 分钟 | 12 小时 | 是 |
| Enterprise | 自定义 | 自定义 | 是 |

- 空闲超时：用户无操作超过此时间后自动登出
- 绝对超时：无论是否有操作，超过此时间后强制登出

### 4.3 会话安全措施

- 每个会话绑定设备指纹（User-Agent + 屏幕分辨率 + 时区）
- 检测到设备指纹变化时要求重新验证身份
- Ajax 请求携带 CSRF Token
- 会话 Token 使用 JWT，包含用户 ID、角色、过期时间
- Token 签名使用 RS256 算法，密钥定期轮换

### 4.4 并发会话控制

| 套餐 | 最大并发会话数 |
|------|-------------|
| Free | 1 |
| Business | 3 |
| Enterprise | 无限制 |

超过最大会话数时，最早的会话被自动终止。

### 4.5 会话终止

- 用户主动登出：立即销毁会话
- 超时登出：会话标记为过期，10 分钟后清理
- 管理员强制登出：立即销毁目标用户所有会话
- 密码修改后：所有已存在会话失效

## 5. 风险等级定义

### 5.1 用户风险等级

| 风险等级 | 触发条件 | 限制措施 |
|---------|---------|---------|
| low | 正常使用用户 | 无额外限制 |
| medium | 触发以下任一条件：1 次支付失败、1 次账号异常登录、1 次工单争议 | 增加两步验证提示、API 调用限额降为 80% |
| high | 触发以下任一条件：3 次支付失败、账号曾被冻结、涉及退款争议、疑似欺诈 | 强制两步验证、API 调用限额降为 50%、部分高敏感操作需审批 |
| critical | 触发以下任一条件：确认欺诈行为、法律要求冻结、严重违反服务条款 | 账号冻结、所有操作需人工审核 |

### 5.2 操作风险等级

| 操作 | 风险等级 | 审批要求 |
|------|---------|---------|
| 修改密码 | low | 无 |
| 修改邮箱 | medium | 需验证新旧邮箱 |
| 修改手机号 | medium | 需验证新旧手机号 |
| 修改套餐 | medium | 需确认操作 |
| 退款 >= $100 | high | 需财务总监审批 |
| 关闭账号 | high | 需确认 + 7 天冷静期 |
| 删除所有数据 | critical | 需双重确认 + 管理员审批 |

### 5.3 风险等级变更流程

- risk_level 存储在 business.customers 表中
- 系统自动检测风险行为并变更等级
- 客服人员可手动调整风险等级（需记录原因）
- 等级变更记入审计日志（business.audit_logs）
- 等级从中等升为高等时自动通知安全团队

## 6. 安全事件响应

### 6.1 安全事件分类

| 级别 | 描述 | 响应时间 | 处理团队 |
|------|------|---------|---------|
| P0 | 数据泄露、系统入侵 | 15 分钟 | 安全团队 + 管理 |
| P1 | 大规模账号异常、DDoS 攻击 | 30 分钟 | 安全团队 |
| P2 | 单个账号被入侵、异常登录 | 2 小时 | 客服 + 安全 |
| P3 | 可疑活动、安全预警 | 24 小时 | 客服 |

### 6.2 事件处理流程

1. 自动检测或用户报告安全事件
2. 系统自动触发预定义响应（如冻结账号）
3. 安全团队评估事件级别
4. 按响应时间要求介入处理
5. 记录处理过程和结果
6. 事后复盘并更新安全策略

## 7. 数据访问审计

### 7.1 审计日志

所有敏感操作（登录、密码修改、数据导出、角色变更等）均记录在 business.audit_logs 表中：

- audit_id：唯一标识
- user_id：操作用户
- action：操作类型
- resource_type：资源类型（customer、order、ticket、approval）
- resource_id：资源 ID
- old_value / new_value：变更前后的值
- trace_id：关联的请求追踪 ID
- created_at：操作时间

### 7.2 日志保留

- 审计日志保留 2 年
- 超过 2 年的日志归档至冷存储
- Enterprise 客户可申请延长日志保留期限至 7 年

### 7.3 日志查询权限

| 角色 | 可查看的日志范围 |
|------|---------------|
| support_agent | 自己的操作日志 |
| manager | 团队所有成员的操作日志 |
| admin | 全部操作日志 |

## 8. 合规认证

- CloudDesk 遵循 SOC2 Type II 安全控制标准
- 数据处理符合 GDPR 要求（详见数据隐私政策）
- 支持客户签订 DPA（数据处理协议）
- 每年进行第三方安全审计
